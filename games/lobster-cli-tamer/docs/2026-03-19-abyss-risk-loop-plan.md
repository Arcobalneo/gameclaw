# 2026-03-19 深渊风险循环优化方案（plan）

## 背景

当前深渊风险模型存在一个结构性漏洞：

- 只有 **精英层** 会调用 `_apply_plague()`
- 退出深渊时，只有 **已感染疫病** 的虾米才会触发死亡结算
- 结果是：玩家如果只在 **低层普通层反复 farm**，理论上可以长期获得深渊掉落，却几乎不承担退出病死风险

这会直接削弱预期中的核心 loop：

1. 大世界 farm / 抓宠 / 组队
2. 深渊作为数值验证与风险挑战
3. 深渊掉落词条材料
4. 用材料增强队伍
5. 再去挑战更高层数

当前漏洞会把第 2 步变成“可被低层安全刷穿的资源点”，从而让深层挑战失去驱动力。

---

## 设计目标

### 必须达成

- **进入深渊本身就应带来风险，而不是只有碰到精英才有风险**
- **低层 farm 可以存在，但不能长期零代价**
- **高层深渊必须显著更危险，但也显著更赚**
- 风险应当是**可预期、可理解、可规划**的，而不是纯随机恶心玩家

### 明确不要

- 不要简单改成“所有层都高概率直接得病”
- 不要让玩家第一次浅尝深渊就莫名暴毙
- 不要只靠拉高敌人数值解决 loop 问题
- 不要让系统从“可管理风险”退化成“纯赌脸惩罚”

---

## 推荐方案："深渊污染（taint） + 疫病（plague）" 两层压力模型

### 核心思路

把现在“是否感染疫病”这一层，拆成两层：

1. **深渊污染（taint）**：进入深渊后的基础累积压力
   - 普通层也会累积
   - 本身不直接触发死亡掷骰
   - 但会把“低层无脑 farm”变成有成本的行为

2. **深渊疫病（plague）**：高风险状态
   - 由精英/Boss/高污染转化触发
   - 退出深渊时进入死亡结算
   - 仍保留当前“高压深渊”的戏剧性

### 为什么推荐这个方案

因为它同时解决两件事：

- **低层 farm 不再绝对安全**：哪怕不碰精英，也会积累污染
- **高层依然更危险**：精英 / Boss / 长局爬塔会更快把污染转成真正疫病

也就是说，风险的来源从“有没有碰到精英”改成“你是否持续深入深渊”。

---

## 具体规则（建议版）

### A. 新状态：`abyss_taint`

给每只虾米增加一个新字段：

- `abyss_taint: int = 0`

含义：

- 表示这只虾受到了多少“深渊污染”
- 污染不会立刻致死，但会推动其向疫病状态演化

### B. 每层通关后的污染累积

建议规则：

- **普通层通关**：全体存活出战虾 `abyss_taint +1`
- **精英层通关**：全体存活出战虾 `abyss_taint +2`
- **Boss 层通关**：全体存活出战虾 `abyss_taint +3`

说明：

- 这样一来，哪怕只刷普通层，污染也会增长
- 精英 / Boss 会明显加速污染压力

### C. 污染转疫病阈值

建议阈值：

- 当某只虾 `abyss_taint >= 3` 时：
  - 自动转为 `has_plague = True`
  - `plague_floors += abyss_taint`
  - `abyss_taint = 0`

含义：

- 浅层 farm 1~2 层，不会立刻病死
- 但如果反复刷，污染会累计并最终转成真正疫病
- 这样风险是**延迟出现、但不可无限逃避**

### D. 精英 / Boss 保留“直接高压”特性

在保留当前精英层特色的基础上，建议改为：

- **精英层通关**：
  - 继续保留当前“深渊疫病事件”的强压感
  - 但不再是唯一感染源
  - 推荐：
    - 若未感染：`abyss_taint +2`
    - 若已感染：`plague_floors +1`

- **Boss 层通关**：
  - 额外强化压力
  - 推荐：
    - 若未感染：直接转疫病，`plague_floors += 2`
    - 若已感染：`plague_floors +2`

目的：

- 精英 / Boss 仍然是风险拐点
- 但“普通层 0 风险”这个漏洞被彻底堵上

### E. 退出深渊时的结算

保留当前大框架：

- 只有 `has_plague` 的虾会掷死亡骰
- 死亡率仍参考 `plague_floors * plague_rate_per_floor`，并受上限限制

但要增加一个附加结算：

- **未转疫病的 `abyss_taint` 不清零，直接带出深渊保留**

这一步非常关键。

原因：

- 如果退出时直接清掉污染，低层 farm 又会恢复成“打一层就跑，永远安全”
- 只有污染会跨 run 留存，玩家才必须认真对待“反复浅刷”的长期成本

---

## 与现有 loop 的耦合方式

### 1. 大世界 farm 的价值被强化

因为：

- 深渊不再适合无脑刷低层
- 玩家会更愿意先去大世界补图鉴、补阵容、找更强个体
- 然后带更好的队伍进深渊，而不是把深渊当成零风险素材井

### 2. 深渊材料收益要继续深度绑定层数

建议在后续平衡里同步强化：

- 普通低层：保底少量材料
- 中层精英 / Boss：更高材料密度
- 深层：更高阶材料 + 更高数量 + 稀有网

也就是：

- **浅层可补一点，但不值得长期刷**
- **真正想高效拿材料，必须承受更深层风险**

### 3. 净疫盐 / 治疗系统要纳入闭环

建议把 `cure_plague` 的设计目标，从“偶发救急道具”升级为“循环调压阀”：

- 可以清除 `has_plague`
- 也应该清除 `abyss_taint`
- 否则新污染系统会缺少明确的排解出口

---

## 为什么不推荐以下简单方案

### 方案 X：所有普通层也直接有概率感染疫病

问题：

- 太随机
- 玩家感知会变成“怎么突然又病了”
- 很难区分“浅层试探”与“深层挑战”的风险档位

### 方案 Y：每层退出都直接按层数掷病死

问题：

- 太硬
- 浅层体验容易变成劝退
- 缺少逐步升温的风险曲线

### 方案 Z：只把精英层出现频率调高

问题：

- 本质上没修 loop
- 玩家仍然可以利用低层和撤退时机规避真正风险

## 实施顺序（推荐）

### Phase 1：结构修复（优先）

目标：先堵住“低层无限安全 farm”漏洞。

Checklist：

- [x] 给 `Creature` 增加 `abyss_taint`
- [x] 序列化 / 反序列化 `abyss_taint`
- [x] 深渊每层胜利后为存活成员增加污染
- [x] 实现“污染转疫病”逻辑
- [x] 退出深渊时保留未清除污染
- [x] `cure_plague` 同时清除污染
- [x] UI / 详情页展示污染与疫病状态

### Phase 2：收益再平衡（随后）

目标：让风险与收益真正匹配。

Checklist：

- [ ] 按层数重调材料掉落曲线
- [ ] 提高中高层的单位时间收益
- [ ] 让浅层收益足够低，只适合过渡，不适合刷穿

### Phase 3：验证与观测

Checklist：

- [ ] 脚本模拟 50 / 100 / 200 次 abyss runs
- [ ] 统计浅层 farm 的平均收益 / 污染增长 / 病死率
- [ ] 统计中层 / 深层的风险收益比
- [ ] 用真实 agent 跑一轮，看是否仍会选择“无限刷低层”

## 最终建议

**我推荐不要粗暴地把“普通层 = 也会直接得病”当成最终答案。**

更合理的修法是：

- 让**普通层也会积累污染**
- 让污染在多次浅刷后**不可避免地转化为疫病压力**
- 让精英 / Boss 成为污染快速升级的风险拐点
- 让治疗道具承担“清压阀”的闭环作用

这样改完后，玩家仍然可以：

- 浅层试探
- 低风险过渡
- 小规模 farm

但无法：

- 永久零代价刷低层
- 把深渊当作没有长期成本的素材井

这会更符合这个游戏想要的核心循环，而不是只修一个表面漏洞。

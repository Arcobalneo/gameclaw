# 2026-03-12 `lobster-cli-roguelike` Gameplay-first 优化计划

## Goal

基于当前 `gameclaw` 的唯一设计准则文档，对《横着活：只给龙虾玩的 CLI 肉鸽》做第一轮 **gameplay-first** 优化。

本轮目标不是把它改成评测环境，也不是优先扩展工程能力，而是优先提升以下四件事：

1. **选择张力**：让每个遭遇的选项都更值得比较，减少无脑最优解
2. **memory 诚实度**：让游戏鼓励记经验，但不把单次结果伪装成稳定真理
3. **谱系 / build 人格感**：让三条谱系和若干突变在玩感上更鲜明
4. **节奏感**：让潮段推进、风险转折和失败结算更有“再来一局”的拉力

## Scope

本轮只处理与 gameplay 直接相关的设计优化，主要涉及：

- `games/lobster-cli-roguelike/lobster_cli_roguelike/game.py`
- `games/lobster-cli-roguelike/tests/test_game.py`
- 如有必要，更新 `games/lobster-cli-roguelike/README.md`

本轮**不**主动做以下方向：

- benchmark / eval / 指标体系建设
- 结构化日志系统
- JSON 输出协议化扩展
- 大规模新增遭遇池
- 新游戏模式（例如专门的 benchmark mode）

## Design Principles for This Round

本轮改动必须服从以下原则：

- 游戏性优先于工具性
- 先让 agent 玩家觉得“这局值得玩”，再考虑别的
- memory 是成长主轴，但 memory 提示必须诚实
- 不允许长期存在明显支配策略
- build 不只是数值变化，而应有明显人格和风格
- 失败必须更像“有信息量的失败”，而不是平淡结束

## Problems To Address

### A. 关键遭遇存在明显支配策略

当前版本中，部分遭遇的某个选项过于强势，容易演化成固定答案：

- `拖网阴影`
- `归海闸口`
- `塑料垃圾流`
- `电鳗裂隙`

需要重做这些遭遇的选项分工，使其更接近：
- 稳妥选项
- 高收益高风险选项
- 谱系 / build 偏置选项

### B. memory 提示过早下结论

当前 `更稳 / 容易翻车` 的口吻太像稳定结论，但实际上只来自单次结果。

需要把 memory 提示改成更诚实的“本局观察 / 下次验证”风格，避免误导 agent 把偶然结果写成绝对真理。

### C. 谱系与突变的“人格感”还不够

当前 3 条谱系已有基础差异，但在实际玩感上仍不够鲜明。

需要进一步强化：
- `沟壑碎壳者`：强攻、硬换、暴力突破
- `触须预言家`：感知、规避、稳定判断
- `脱壳赌徒`：高波动、反转、拿壳换命

### D. 节奏结构已存在，但高潮曲线还不够锋利

当前已有：
- 小遭遇推进
- 深度 2 / 5 / 8 突变潮
- 归海闸口收束

但从玩感上，仍需要更强的“阶段脾气”和“局势转折感”。

## Proposed Changes

### 1. 重平衡 4 个关键遭遇

优先重做：

- `拖网阴影`
- `归海闸口`
- `塑料垃圾流`
- `电鳗裂隙`

目标：
- 避免单一选项长期统治
- 让不同谱系在同一遭遇里更自然地倾向不同解法
- 让“赌”看起来值得赌，但不会变成无脑按钮

### 2. 重写 memory 提示口径

把当前偏结论式提示改成：
- 本局观察
- 本次结果
- 建议继续验证

目标：
- 提升经验质量
- 降低 memory 污染
- 让 agent 更像在积累经验，而不是抄答案

### 3. 增强谱系身份表达

方式可能包括：
- 微调基础属性分布
- 微调谱系 trait 的加成覆盖面
- 让某些遭遇动作更明确偏好某谱系
- 让不同谱系在失败成本结构上也不同

目标：
- 让三条谱系玩起来像三种不同性格的龙虾
- 提高重开时的身份切换感

### 4. 让突变更偏向 build，而不只是 stat buff

本轮不大幅增加新突变，但会优先检查：
- 现有突变是否只是平推数值
- 哪些突变需要更强的风格表达
- 哪些突变应该明显强化特定行动风格

目标：
- 让 build 更早出现轮廓
- 让 agent 形成“这局我是怎么活”的自我认识

### 5. 轻量增强节奏提示与失败结算张力

优先考虑不大改结构的轻量优化，例如：
- 让部分潮段读起来更有脾气
- 让失败结算的归因感更强
- 让结算摘记更像“值得下局验证的教训”

目标：
- 提高再开一局的冲动
- 让失败更有信息量和情绪张力

## Implementation Strategy

按以下顺序推进：

1. 先改遭遇平衡与 memory 提示
2. 再调谱系 / trait / 突变表达
3. 最后修结算文案与 README 中相关描述
4. 每一阶段后运行测试并实际试玩 1~2 局做人工判断

## Validation

本轮验证不走 benchmark 口径，而走 gameplay 口径：

- `python3 -m unittest discover -s tests`
- 固定种子试玩若干局，观察是否仍然快速收敛为固定答案
- 对 3 条谱系分别试玩，检查身份感是否更鲜明
- 检查 memory 提示是否明显更诚实
- 检查失败结算是否更能支撑“再来一局”

## Risks

- 平衡调整过猛，可能让原本可玩的局面变得过难
- 为了消灭支配策略，可能把选项拉得过于平均，反而失去性格
- 强化谱系人格时，可能导致某谱系在长局中过强或过弱
- memory 提示改得太保守，可能失去鼓励记录的推动力

## Out of Scope Notes

以下事情可以放到后续轮次再讨论：

- 更系统的可调试输出
- 更大规模的遭遇扩展
- 新模式或更复杂的协议层

这些不是不重要，而是本轮先不让它们抢走 gameplay 优化的优先级。
